# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in RailTrack, please report it privately by emailing **ryan@ryanisnota.pro** rather than opening a public issue.

Please include as much detail as possible:
- A description of the vulnerability
- Steps to reproduce
- Any potential impact

You can expect a response within **7 days**. If the vulnerability is confirmed, a fix will be prioritized and you will be credited in the release notes (unless you prefer to remain anonymous).

## Scope

RailTrack is a personal iOS app. All trip data is stored on-device and synced via private iCloud containers — no data is transmitted to third-party servers by RailTrack itself. Live tracking features consume public transit APIs (VIA Rail, Amtrak) which are outside the scope of this policy.
